# ADCIRC mpi.sh 수행 전 진단 우선순위

`mpi.sh` 실행 전 **P1-a ~ P6 전 항목을 병렬로 동시 진단**한다.  
진단 완료 후 결과를 일괄 수집하여 FAIL 여부를 판정하고 수행 여부를 결정한다.

**우선순위 번호의 의미**: 진단 실행 순서가 아닌, FAIL이 여러 개일 때 **수정 순서**.  
P1-a부터 순서대로 고치면 하위 FAIL이 연쇄 해소되는 경우가 많다.

---

## P1-a — 필수파일 존재 확인 `즉시 종료형`

파일 하나라도 없으면 adcprep 또는 mpirun이 아무 의미 있는 오류 없이 죽는다.

| 체크 항목 | 판정 기준 | 실패 시 결과 |
|-----------|-----------|-------------|
| `adcprep` | 존재 + 실행권한 (`-x`) | adcprep --partmesh 즉시 실패 |
| `padcirc` | 존재 + 실행권한 (`-x`) | mpirun 즉시 실패 |
| `fort.14` | 존재 + 크기 > 0 | adcprep 격자 읽기 실패 |
| `fort.15` | 존재 + 크기 > 0 | adcprep 제어파일 파싱 실패 |
| `machine` | 존재 | mpirun 노드 할당 실패 |

---

## P1-b — NP 일관성 확인 `즉시 종료형`

세 곳의 NP 값이 **모두 일치**해야 한다.  
불일치 시 MPI 할당 오류 또는 adcprep 파티션과 padcirc 실행 코어 수 불일치.

```
mpi.sh   →  adcprep --np  NP   ──┐
mpi.sh   →  mpirun  -n    NP   ──┼── 세 값 일치 필수
machine  →  슬롯 수       ≥ NP ──┘
```

> **확인 방법**
> ```bash
> grep -E "\-\-np|\-n " mpi.sh       # mpi.sh NP 값
> wc -l machine                        # machine 슬롯 수 (노드파일 형식에 따라 다름)
> ```

## P2 — fort.15 가변 파라미터 검증 `설정 오류형`

수행이 시작되거나 이상한 값으로 진행된다.  
아래 파라미터는 태풍·격자 변경 시마다 달라지므로 매번 확인한다.

| 파라미터 | 판정 기준 | 실패 시 결과 |
|----------|-----------|-------------|
| `RNDAY` (총 수행일수) | > 0, **그리고 > `DRAMP`** | DRAMP 종료 전에 수행 끝남 |
| `DRAMP` (램프 기간) | 0.5 ~ 2.0 일 권고 | < 0.25일이면 초기 충격으로 수치 불안정 위험 |
| `DT` (시간간격, 초) | > 0 | 0 또는 음수면 즉시 크래시 |
| `STATIM` (시작 시각, days) | IHOT=0 → `0.0` 권고 | hotstart 시 이전 수행 종료 시각과 불일치 시 불연속 |
| `IHOT` (Hotstart 플래그) | 0, 1, 2 중 하나 | 그 외 값이면 파일 읽기 오류 |

> **fort.15 파라미터 위치 (코멘트 제거 기준 순서)**
> ```
> 1  : RUNDES
> 2  : RUNID
> 3  : NFOVER
> ...
> 6  : IHOT
> ...
> 13 : NWP  (이후 NWP개 속성명 줄 존재)
> ...      (NWP 속성명 수만큼 줄 이동)
> +3 : NWS
> +5 : DT
> +6 : STATIM
> +7 : REFTIM
> +8 : RNDAY
> +9 : DRAMP
> ```

## P3 — 조건부 필수파일 존재 확인 `NWS·NWP·IHOT 의존형`

fort.15 플래그 값에 따라 필요 파일이 달라진다.  
없으면 수행 시작 직후 파일 읽기 오류로 종료.

### NWS (바람장 방식)

| NWS 값 | 방식 | 필요 파일 |
|--------|------|-----------|
| `0` | 바람장 없음 (조위 전용) | 없음 |
| `±8` | OWI 격자 바람장 | `fort.221` (기압), `fort.222` (바람) |
| `±19` | GAHM Best Track | `fort.22` |
| `±20` | GEOSR 태풍 트랙 ← **현재 사용** | `fort.22` |

### NWP (가변 속성)

| 조건 | 필요 파일 | 추가 확인 |
|------|-----------|-----------|
| `NWP > 0` | `fort.13` 존재 | fort.15 속성명 ↔ fort.13 속성명 **정확히 일치** |
| `NOLIBF = 2` (Manning's) | `fort.13` 존재 | `mannings_n_at_sea_floor` 속성 포함 여부 |

### IHOT (Hotstart)

| 조건 | 필요 파일 |
|------|-----------|
| `IHOT = 0` | 없음 (콜드스타트) |
| `IHOT = 1` | `fort.67` |
| `IHOT = 2` | `fort.68` |

## P4 — 바람장–모의기간 일치 확인 `기간 불일치형`

수행은 시작되지만 바람장 데이터가 끊기는 시점에서 강제 종료되거나  
이후 구간이 바람 = 0으로 계산된다.

```
fort.22 첫 트랙 시각    ≤  STATIM
fort.22 마지막 트랙 시각 ≥  STATIM + RNDAY
```

### GEOSR fort.22 포맷 (NWS=20)

```
YYYYMMDDHH  0  0  위도×10  경도×10  중심기압(hPa)
2025090100  0  0  285      1255     0970
2025090106  0  0  300      1263     0960
...
```

### 추가 확인 항목

| 항목 | 판정 기준 | 이유 |
|------|-----------|------|
| 트랙 시각 간격 | 균일 (보통 6시간) | 불규칙하면 내부 선형 보간 오류 |
| 중복 시각 | 없어야 함 | 동일 시각 2회 이상 존재 시 보간 오동작 |
| 역순 시각 | 없어야 함 | 시각이 감소하면 즉시 오류 |

> **시각 단위 주의**  
> fort.15 `RNDAY`는 일수(days), fort.22는 `YYYYMMDDHH` 절대 시각.  
> 비교 시 기준 날짜(REFTIM)를 통해 변환 필요.

## P5 — Hotstart 타임스탬프 일치 `IHOT > 0인 경우만`

IHOT=0(콜드스타트)이면 이 항목은 건너뛴다.

```
fort.67 또는 fort.68 의 마지막 타임스탬프  ==  fort.15 의 STATIM
```

불일치 시 수행은 시작되지만 이전 수행과 시각 불연속 → **결과 신뢰 불가**.

> **확인 방법**  
> fort.67/68은 바이너리이므로 직접 열람 대신 생성 시각 또는  
> 이전 수행의 `NHSTAR`, `NHSINC` 설정으로 기록된 타임스탬프 로그를 참조.

## P6 — CFL 안정성 검토 `수치 발산형`

수행은 완료되지만 시간이 지나면서 수치가 발산한다.  
`log.dat`에서 뒤늦게 발견되므로 사전 확인이 중요.

### ADCIRC GWCE 기준 (반묵시적)

```
0  <  C = sqrt(g × h_max) × DT / dx_min   ≤  4  권고
```

> **C > 0 필수 조건**: DT · h_max · dx_min 이 모두 양수여야 한다.  
> 셋 중 하나라도 0 이하면 CFL 자체가 무의미하므로 수행 전 즉시 중단.

### 격자 해상도별 권고 DT

| 격자 해상도 | 대표 최대 수심 | 권고 최대 DT | DT=2s 시 C값 |
|------------|--------------|------------|-------------|
| 100 m | 50 m | ≤ 5 s | **0.44 ✔** |
| 100 m | 100 m | ≤ 3 s | **0.63 ✔** |
| 100 m | 200 m | ≤ 2 s | **0.89 ✔** |
| 500 m | 50 m | ≤ 22 s | — |
| 1 km | 100 m | ≤ 45 s | — |

> **현재 설정** (G_100m, DT=2s): 정상 범위

### log.dat에서 CFL 초과 징후

```bash
grep -E "EXCEEDING STABILITY|MAXIMUM WATER SURFACE ELEVATION EXCEEDS" log.dat
```

## 진단 흐름 요약

```
                    ┌─────────────────────────────────────┐
                    │         병렬 진단 (동시 실행)          │
                    │                                     │
                    │  P1-a  필수파일 존재                  │
                    │  P1-b  NP 일관성                     │
                    │  P2    fort.15 가변 파라미터          │
                    │  P3    조건부 필수파일                 │
                    │  P4    바람장–모의기간 일치            │
                    │  P5    Hotstart 타임스탬프 (IHOT>0)  │
                    │  P6    CFL 안정성                    │
                    └──────────────┬──────────────────────┘
                                   │ 전체 결과 수집
                    ┌──────────────▼──────────────────────┐
                    │           판정                       │
                    │                                     │
                    │  FAIL 있음?  ──YES──▶  수행 중단     │
                    │                        (FAIL 목록    │
                    │                         전체 출력)   │
                    │  WARN만?    ──YES──▶  경고 출력 후   │
                    │                        수행 진행     │
                    │  모두 PASS  ──────▶   수행 진행      │
                    └──────────────┬──────────────────────┘
                                   │
                                   ▼
                               mpi.sh 수행
```

> **우선순위의 의미**: 진단은 병렬로 수행하되, FAIL이 여러 개일 때  
> **수정 순서**는 P1-a → P1-b → P2 → … → P6 순으로 한다.
