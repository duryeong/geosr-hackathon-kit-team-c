# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
태풍 통보문 수신 → GEO-ADCIRC(ADCIRC+SWAN) 침수모형 수행 → 결과 가시화 → 위험도 판단 → 의사결정 보고서 작성. 담당자가 단계마다 수동 개입하던 전 과정을 통합 자동화.

## Before (기존 수작업 절차)
1. 기상청 태풍 통보문이 오면 담당자가 인지 → 케이스 폴더를 수행 위치로 수동 복사.
2. 전처리(바람장·hotstart·조위) → 모델 본수행 → 조위 → 후처리를 단계별로 수동 실행/대기.
3. (레거시 crontab은 경로·단계가 깨져 실질적으로 손이 감: `/home/storm/2022` 하드코딩, onlytide 누락, /tmp 상태관리.)
4. 모델 결과 가시화(FigureGen/GMT)를 사람이 실행.
5. 가시화·수치를 보고 사람이 위험도 판단.
6. 의사결정 보고서를 사람이 직접 작성.

## After (에이전트화된 절차)
1. `check_typhoon.sh`(crontab 5분)가 신규 통보문 케이스를 자동 감지(영속 상태파일로 중복 방지).
2. `run_pipeline.sh`가 01_pre→02_model→03_onlytide→04_post를 정정된 순서로 자동 오케스트레이션 + 타임스탬프 로깅.
3. (예정) 가시화 산출물 → 의사결정 에이전트(위험도 판단) → 보고서 초안 자동 작성.
